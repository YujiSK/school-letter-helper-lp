# デプロイ・実機検証チェックリスト：School Letter Helper

本書は、「School Letter Helper」のNext.jsモックプロトタイプをVercel等の本番サーバーにデプロイし、実機検証（10人テスト）を安全かつ効果的に実施するためのチェックリストおよび案内資料である。

---

## 1. デプロイ前確認 (Pre-deployment Checklist)

デプロイを開始する前に、以下の項目が完了しているか確認してください。

- [x] **Git 最新化**: ローカルのすべての変更（ESLint修正など）がコミットされ、リモートのGitHub（`main` ブランチ）に push されていること。
- [x] **ビルドチェック**: ローカル環境で `npm run build` がエラー・警告なしで成功していること。
- [x] **APIのモック状態確認**: `app/api/analyze/route.ts` が、外部API（Vision API, LLM API）に接続せず、1.5秒の遅延ののちに模擬データを返す仕様になっていること。
- [x] **シークレットキーの不在確認**: 環境変数ファイル（`.env.local` 等）やソースコード中に、不要なAPIキーや認証情報が書き込まれていないこと。

---

## 2. Vercel環境での動作確認項目

Vercelにデプロイ完了後、公開されたブラウザURL上で以下の挙動を確認してください。

- [ ] **SSL/HTTPS**: `https://` で安全に接続され、警告が出ないこと。
- [ ] **モックAPIレスポンス**: アップロード後、ローディング状態が走り、正常に結果画面へ遷移すること。
- [ ] **多言語切り替え**: 各画面で言語（日本語・英語・ポルトガル語・ベトナム語）が正しく切り替わり、文字化けがないこと。
- [ ] **LocalStorage**: 
  - ToDoのチェック状態を変更して再ロードしても状態が維持されること。
  - ホームに戻った際に履歴カードが表示されること。
  - 「すべての履歴を削除」を押してダイアログを承認後、履歴が消去されること。
- [ ] **sessionStorage (保存しない設定)**:
  - 画面1で「この端末に履歴を保存しない」をオンにして画像を送信。
  - 結果画面・返信画面の操作は正常に行えること。
  - ホームに戻った（またはブラウザを閉じた）時に、履歴一覧に今回のレコードが**一切残っていない**こと。

## 3. スマホ実機テストチェックリスト (Mobile Verification)

iPhone (Safari/Chrome) や Android (Chrome) の実機で、以下のモバイル固有の挙動を確認してください。

- [ ] **カメラ起動**: アップロードボタンをタップした際、スマートフォンのカメラが起動してその場で写真を撮影できる、あるいはアルバムから画像を選択できること。
- [ ] **レスポンシブ崩れの有無**: 320px（iPhone SE等）や390px幅で、文字の折り返しやボタンの重なりがなく、フッターアクションエリアが画面内に綺麗に収まっていること。
- [ ] **返信文のコピー**: 「コピーする」ボタンをタップした際、正しくクリップボードに日本語の文章が保存され、かつ「コピーしました！」のトースト通知が数秒間表示されること。
- [ ] **コピー文の貼り付け**: メモ帳やLINE、連絡帳アプリなどの外部アプリへ、コピーした文章を正常に貼り付けできること。
- [ ] **LocalStorageの挙動**: ブラウザのタブを閉じて開き直しても、前回のToDoのチェックや、設定（お子様の名前・保護者名）が自動入力されたままであること。

## 4. 個人情報保護に関する厳守事項

> [!CAUTION]
> **個人情報を含む画像は絶対に収集しない**
> 本検証（10人テスト）は、あくまでUI/UXの操作性や「やることリスト化・返信文生成」というアイデアの受容性を確かめるためのものです。
> 個人情報漏洩のリスクをゼロにするため、テストユーザーに対して以下のルールを徹底してください。
> 
> 1. テスト時には、運営から提供する「サンプル用の模擬おたより画像」を優先して使用すること。
> 2. ユーザーが自身のプリント画像を使ってテストを行いたい場合は、**あらかじめ名前、学校名、住所、電話番号、日付などを太い黒ペンなどで完全にマスキング（黒塗り）した状態の画像**を使用すること。
> 3. テストユーザーが送信する画像データそのものは、アンケートフォーム等でも**一切収集（保存）しないこと**。
> 4. サーバーAPIログには、画像本体やOCRされたテキスト情報などを出力しない（/api/analyze 内部で console.log させない）方針を維持すること。
