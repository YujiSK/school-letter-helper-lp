# MVP要件定義書：School Letter Helper

本書は、外国人保護者向けの学校おたより・ToDo化・返信支援サービス「School Letter Helper」の最小実行可能製品（MVP）における要件定義書である。

---

## 1. システム概要と基本設計思想

### 1.1 プライバシー優先・サーバーDB保存なし設計
本製品の最も重要な検証対象は「外国人保護者に安心して使ってもらうこと」である。学校プリントには子どもの名前、学校名、住所などの機微情報が多数含まれるため、初期MVPでは以下のセキュリティ設計を徹底する。
*   **ステートレス解析**: アップロード画像およびOCRテキストは、当サービスのサーバーDBやファイルストレージには保存しない。ただし、OCR・AI解析のために外部APIへ一時的に送信される。送信先APIのデータ保持ポリシーを確認し、MVP段階では個人名・学校名・住所などの機微情報を可能な限りマスキングしてから解析する。
*   **完全クライアントサイド保存**: 解析結果はブラウザのLocalStorageに保存する。ただし、共有端末では他者に閲覧される可能性があるため、「この端末に保存しない」設定、および「履歴をすべて削除」ボタンを設置する。

### 1.2 ターゲット環境
*   **Webアプリケーション（モバイルファースト）**: インストール不要でXのリンクなどから直接スマホで起動できるレスポンシブWebサイト。

## 2. 画面一覧とユーザージャーニー

MVPは最小限の4画面で構成する。

```
[1. ホーム/アップロード画面] 
       ↓ (カメラ撮影 / 画像選択)
[2. 解析中・ローディング画面]
       ↓ (解析完了)
[3. ToDo・やさしい日本語・翻訳確認画面]
       ↓ (返信文作成ボタン)
[4. 先生宛て返信文生成画面]
```

### ① ホーム / プリントアップロード画面 (Home/Upload)
*   **機能**: 
    *   サービスの価値を伝えるシンプルなヘッダー。
    *   言語切り替えリンク（日本語 / English / Português）。
    *   大きな「プリントを撮影する / 画像を選ぶ」ボタン。
    *   プライバシーに対する安全宣言（「画像はサーバーに保存されません」）の明記。

### ② 解析中・ローディング画面 (Processing/Loading)
*   **機能**:
    *   OCRスキャンとAI要約を実行している間のスケルトンスクリーン、または進捗プログレスバー。
    *   外国人ユーザーが安心できる「読み込んでいます (Processing...)」の多言語テキスト表示。

### ③ ToDo・やさしい日本語・翻訳確認画面 (Result/Dashboard)
*   **機能**:
    *   **やさしい日本語要約表示**: プリントの要旨を3文以内の箇条書きで平易に表現。
    *   **翻訳表示**: ユーザーが選択した母語（英語・ポルトガル語など）による要約の併記。
    *   **やること（ToDo）リスト**: 自動抽出された締切日、提出物、持ち物、金額のチェックリスト（チェック状態はLocalStorageで保持）。
    *   **「返信文を作る」ボタン**: 画面④へ遷移。

### ④ 先生宛て返信文生成画面 (Reply Generator)
*   **機能**:
    *   **返信テンプレート選択**: 「欠席連絡」「行事の参加」「面談日程の調整」「質問・相談」の簡易ラジオボタン。
    *   **生成された返信文（日本語）**: 先生に対して失礼のない自然な日本語テキストを出力。
    *   **「コピーする」ボタン**: クリップボードへコピーし、既存の連絡アプリやメールに貼り付け可能にする。

## 3. 機能要件 (Feature Requirements)

### 3.1 コア機能モジュール
1.  **カメラ連携・画像選択機能**
    *   スマートフォンのカメラ起動、またはアルバムからの画像（JPEG/PNG/PDF）の選択。
2.  **OCR（テキスト抽出）連携機能**
    *   Google Cloud Vision API 等を利用し、縦書き・手書き・低解像度が含まれるプリント画像から高精度に日本語テキストを抽出する。
3.  **AI構造化解析（LLM連携）機能**
    *   抽出された日本語テキストをもとに、以下のJSONデータを生成する。
        *   `summary`: やさしい日本語による全体要約（箇条書き）
        *   `deadlines`: 締切日および対応が必要なアクション
        *   `belongings`: 当日必要な持ち物・提出物の一覧
        *   `cost`: 必要な費用・集金金額
        *   `needs_reply`: 先生への返信が必要かどうかの真偽値
4.  **多言語自動翻訳機能**
    *   構造化されたJSONデータを、ユーザーのブラウザ言語または選択言語（英語/ポルトガル語）に自動翻訳する。
5.  **返信文自動下書き生成機能**
    *   指定された文脈（例：欠席理由など）とプリントの内容を掛け合わせ、丁寧な日本語の連絡文を生成する。

## 4. API設計 (API Endpoints)

サーバー側DBを持たないため、APIはステートレスな解析エンドポイント1本のみで運用する。

### 4.1 `POST /api/analyze` (おたより画像解析)
*   **Request Body**:
    *   `image`: Base64エンコードされた画像データ
    *   `targetLang`: 翻訳先言語コード（`en`, `pt`, `ja-easy`）
*   **Processing**:
    1.  画像をOCR APIへ送信し、日本語のプレーンテキストを抽出。
    2.  抽出したテキストをLLM (Gemini 1.5 Pro / GPT-4o-mini) プロンプトに入力。
    3.  プロンプトによって、やさしい日本語化、タスクのJSON構造化、指定言語への翻訳、返信文テンプレートの生成を一括で行う。
3.  **Response (JSON)**:
    ```json
    {
      "success": true,
      "originalText": "OCRで抽出された生の日本語文章...",
      "easyJapaneseSummary": [
        "6がつ12にち までに だしてください。",
        "えんそくに いきます。",
        "みずとう、ぼうし、あまぐを もっていきます。"
      ],
      "translatedSummary": [
        "Submit by June 12.",
        "Going on a school trip.",
        "Bring a water bottle, hat, and rain gear."
      ],
      "todos": [
        { "id": 1, "task": "Submit consent form", "deadline": "June 12" },
        { "id": 2, "task": "Prepare items: Water bottle, hat, rain gear", "deadline": "Event day" }
      ],
      "replyTemplates": {
        "attendance": "いつもお世話になっております。遠足の件、参加いたします。提出用紙は明日持たせます。よろしくお願いいたします。",
        "absence": "いつもお世話になっております。遠足の件、今回は欠席とさせていただきます。よろしくお願いいたします。"
      }
    }
    ```

## 5. 技術スタック (Tech Stack)

*   **フレームワーク/フロントエンド**: 
    *   Next.js (App Router)
    *   Tailwind CSS (UIスタイリング)
*   **バックエンド (API Routes)**:
    *   `/api/analyze` エンドポイント (APIキーの隠蔽と、Vision API / LLM APIのプロキシサーブ)
*   **外部サービス (API)**:
    *   OCR: Google Cloud Vision API（縦書き日本語の読み取り精度が最も高いため）
    *   LLM: OpenAI GPT-4o-mini または Gemini 1.5 Flash (低コスト・高速レスポンス重視)
*   **クライアントストレージ**:
    *   LocalStorage
*   **デプロイ・ホスティング**:
    *   Vercel または Cloudflare Pages

## 6. セキュリティ・プライバシー要件

### 6.1 サーバー保存禁止
- アップロード画像をサーバーDBに保存しない
- OCRテキストをサーバーDBに保存しない
- 解析結果をサーバーDBに保存しない
- 一時ファイルを作成する場合は処理完了後に即時削除する

### 6.2 外部API送信時の注意
- OCR処理のため画像を外部APIへ送信する
- LLM処理のためOCRテキストを外部APIへ送信する
- API送信前に、可能な範囲で児童名、学校名、住所、電話番号をマスキングする
- 利用画面に「AI解析のため外部APIへ一時送信されます」と明記する

### 6.3 クライアント保存
- 解析結果はLocalStorageに保存する
- ユーザーが保存しない選択をできるようにする
- 履歴削除ボタンを設置する
- 共有端末では保存しないよう注意文を表示する

### 6.4 ログ設計
- APIログに画像本文、OCR全文、翻訳結果、返信文を出力しない
- エラーログにはステータスコード、処理時間、失敗理由のみを記録する
- 個人情報を含む可能性がある値をconsole.logしない

### 6.5 MVP利用規約上の注意
- 本サービスは学校・自治体の公式連絡手段ではない
- AI解析結果は誤る可能性がある
- 重要な内容は学校・先生に確認する
- 緊急時・医療・法的判断には使用しない
